# Phase 2 — Commercialization System Roadmap

**Status:** Draft roadmap  
**Applies to:** PMD_001–PMD_005 (all approved and frozen)  
**Goal:** Build once, reuse across all five programs  
**Documents to produce:**
1. Commercial Sheets
2. Workbook Templates
3. Facilitator Guide Templates
4. CPD Documentation

---

## 1. Guiding principles

| Principle | Meaning |
|---|---|
| **Build once, reuse everywhere** | Every asset is designed for all five programs, not one-off documents. |
| **PMD-driven** | Commercialization assets derive directly from the approved PMDs. No new program content is created. |
| **Modular** | Each asset has a master template and program-specific inserts. |
| **Client-ready** | Commercial sheets and CPD documents are suitable for prospects, buyers and accreditors. |
| **Facilitator-safe** | Facilitator guides are detailed enough that a prepared substitute can deliver the program. |
| **Version-controlled** | Every document carries version, date and source PMD reference. |

---

## 2. Priority sequence and creation order

| Phase | Order | Workstream | Why first? |
|---|---|---|---|
| **A — Foundation** | 1 | **Workbook Templates** | Defines participant-facing content, visual style and structure used by Commercial Sheets and Facilitator Guides. |
| **A — Foundation** | 2 | **Facilitator Guide Templates** | Converts PMD structure into deliverable instructions; required before pilots. |
| **B — Go-to-market** | 3 | **Commercial Sheets** | Builds on workbook and facilitator assets; used for sales and marketing. |
| **B — Go-to-market** | 4 | **CPD Documentation** | Requires stable workbook and facilitator content; needed for accreditation and B2B sales. |

**Rationale:** Workbooks and facilitator guides are operational foundations. Commercial sheets and CPD docs are external-facing and benefit from completed internal assets.

## 7. Production sequence (detailed)

### Step 1 — Set up the foundation (8h)
1. Create shared folder structure: `AnimaMinds_Commercial_System/`.
2. Create master templates for each workstream.
3. Define variable naming convention.
4. Lock design tokens (colours, fonts, spacing, file naming).

### Step 2 — Produce Workbook Templates (27h)
1. Build master workbook style guide.
2. Create each workbook page template.
3. Generate one full workbook for one program as a pilot (e.g., PMD_001).
4. Review and refine master templates.
5. Adapt for PMD_002–PMD_005.

### Step 3 — Produce Facilitator Guide Templates (20h)
1. Build master facilitator guide structure.
2. Populate scripts and debriefs from PMD module sections.
3. Link each activity to workbook pages.
4. Add assessment and troubleshooting guidance.
5. Adapt for PMD_001–PMD_005.

### Step 4 — Produce Commercial Sheets (18h)
1. Build master commercial sheet layout.
2. Extract promises, outcomes, formats and sector hooks from PMDs.
3. Create program one-pagers.
4. Create comparison matrix and sector sell sheets.
5. Build client FAQ from PMD scope statements and boundaries.

### Step 5 — Produce CPD Documentation (20h)
1. Build CPD pack master.
2. Map learning outcomes to assessments and hours.
3. Format facilitator requirements and evaluation methodology.
4. Add QA process description.
5. Produce per-program packs.

### Step 6 — Review and freeze (12h)
1. Cross-check consistency across all assets.
2. Verify version numbers and PMD references.
3. Pilot with one program (recommended: PMD_004 or PMD_005).
4. Freeze Phase 2 assets.

## 9. Risk register

| Risk | Impact | Mitigation |
|---|---|---|
| PMDs interpreted differently by different authors | High | Every asset references specific PMD section numbers. |
| Visual identity not ready | Medium | Create plain-text/markdown versions first; apply design later. |
| Per-program drift from master template | Medium | Use variables and a single reviewer for consistency. |
| CPD body requires format changes | Medium | Build flexible CPD master with export options. |
| Facilitator guides too long | Low | Keep scripts modular; move extended notes to appendices. |

---

## 10. Success criteria

Phase 2 is complete when:
- [ ] All five programs have a workbook, facilitator guide, commercial sheet and CPD pack.
- [ ] Every external-facing document references the approved PMD version.
- [ ] Master templates exist for each workstream.
- [ ] Assets can be updated from a single master change.
- [ ] A facilitator can deliver any program using only the facilitator guide + workbook.
- [ ] A sales lead can pitch any program using only the commercial sheet.
- [ ] A CPD submission can be prepared from the CPD pack without revisiting PMDs.
