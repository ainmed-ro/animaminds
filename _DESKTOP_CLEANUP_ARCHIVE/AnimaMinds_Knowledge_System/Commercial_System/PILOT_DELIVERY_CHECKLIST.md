# Pilot Delivery Checklist

**Status:** READY FOR PILOT DELIVERY  
**Scope:** Prepare, deliver and debrief a pilot program session  
**Applies to:** PMD_001–PMD_005  
**Version:** 1.0  
**Last updated:** 2026-07-12

---

## Before the pilot

### Program assets

- [ ] Confirm the approved PMD version is frozen and not modified.
- [ ] Use the current Workbook, Facilitator Guide, Commercial Sheet and CPD Package.
- [ ] Verify that v1.0 PMD_001 files (if present) are marked as deprecated.
- [ ] Print or distribute participant workbooks.
- [ ] Prepare slide deck and any required templates or cards.
- [ ] Prepare evaluation instruments from the Feedback Collection Kit.

### Logistics

- [ ] Confirm date, time, location or online platform.
- [ ] Test internet, projector, microphones and breakout rooms.
- [ ] Send joining instructions to participants.
- [ ] Confirm group size is within program limits.
- [ ] Arrange for a second facilitator if needed.

### Facilitator preparation

- [ ] Read the Facilitator Guide for the program.
- [ ] Review the Emotional Safety Protocol (for PMD_002 and PMD_003).
- [ ] Prepare 1–2 backup activities in case a module runs short or long.
- [ ] Prepare a note-taking method for observations and issues.

### Participants

- [ ] Send pre-program survey (T0) at least 48 hours before delivery.
- [ ] Confirm participant roles/sectors for scenario matching.
- [ ] Set expectations about pilot purpose and feedback collection.

---

## During the pilot

### Setup

- [ ] Arrive early / join platform early.
- [ ] Set up materials, whiteboard, shared docs.
- [ ] Welcome participants and explain pilot purpose.

### Delivery

- [ ] Follow the Facilitator Guide module timings.
- [ ] Note any timing overruns or underruns.
- [ ] Capture participant quotes, questions and reactions.
- [ ] Observe engagement, confusion or resistance.
- [ ] Document any technical issues.

### Feedback moments

- [ ] Run a short pulse check at the end of each module.
- [ ] Collect mid-session feedback if the program has a break.
- [ ] Take photos or screenshots of group work (with consent).

### Safety and ethics

- [ ] Apply emotional safety protocol where relevant.
- [ ] Correct any misuse of AI, data or communication examples.
- [ ] Refer beyond-scope issues appropriately.

---

## After the pilot

### Immediate (within 24 hours)

- [ ] Send post-program survey (T1).
- [ ] Collect practical deliverables.
- [ ] Send follow-up email sequence to participants.
- [ ] Compile facilitator debrief notes.

### Within 1 week

- [ ] Review all feedback and deliverables.
- [ ] Identify at least 3 strengths and 3 improvement points.
- [ ] Update the Pilot Evaluation Dashboard.
- [ ] Decide if the program is ready for wider delivery or needs adjustment.

### Within 30 days

- [ ] Send 30-day follow-up survey (T2).
- [ ] Collect testimonials using the Testimonial Collection Kit.
- [ ] Complete the Lessons Learned Template.
- [ ] Present pilot results to stakeholders.
