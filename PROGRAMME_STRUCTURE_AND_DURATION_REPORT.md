# Programme Structure and Duration Report

**Date:** 2026-07-12
**Scope:** All five AnimaMinds programmes

---

## 1. Executive Summary

This report documents the real structure, duration, learning hours, CPD credits and delivery formats found in the source documents. It also records inconsistencies between source documents, the public website and the CMS data model.

**Critical finding:** the public website's programme order does **not** match the stated commercial priority.

## 4. Cross-Document Consistency Issues

### 4.1 "Learning hours" definition is inconsistent

- **PMD Format details table** uses "Learning hours" to mean **contact hours** only.
- **Commercial Sheet / CPD Package** use "Total learning hours" to mean **contact hours + self-study/reflection**.

This means:
- For the same Online format, PMD says 6h while Commercial Sheet says 7h.
- Both are derived correctly from their own definitions, but the label "Learning hours" is ambiguous.

### 4.2 Experience Edition duration conflict

- **PMD** states Experience Edition is **2 days** (one afternoon 3h + one full day 7h = 10 contact hours).
- **Commercial Sheet** states Experience Edition is **2 days** (10h contact + 2h self-study = 12 CPD).
- **User instruction (2026-07-12):** Experience Edition lasts **3 days**.

**Impact:** All source documents and the generated certificate / cover pages need to be updated to reflect 3 days if that is the final decision.

### 4.3 Delivery format names are inconsistent

| Source | Formats listed |
|---|---|
| PMD / Commercial Sheet / CPD Package | Online, La sediul organizației, Open cohort, Experience Edition |
| Public website (`app/programe/page.tsx`) | Online Live, La sediul instituției / organizației, Experience Edition |
| Prisma enum (`DeliveryFormat`) | ONLINE, ONSITE, OPEN_COHORT, EXPERIENCE_EDITION |

**Issues:**
- "Open cohort" appears in source documents and Prisma enum, but not on the public website.
- "La sediul organizației" vs "La sediul instituției / organizației" — minor wording difference.
- "Online" vs "Online Live" — minor wording difference.

### 4.4 One-Pager / generated materials summary line

The generation script (`scripts/generate-programme-materials-v2.ts`) was using a hardcoded summary line:

```
Rezumat: 1 zi (6–8h) sau 2 zile Experience Edition
```

This does **not** match any programme's real data. It was replaced after the audit began with per-format variant lines, but the values were initially based on 1 day / 2 days / Experience Edition and must be reconciled with the final 3-day Experience Edition decision.

### 4.5 Generated certificate CPD values

The certificate and cover pages currently print all three variants:

```
1 day / 1 zi: 6 ore / 6 CPD  |  2 days / 2 zile: 12 ore / 12 CPD  |  Experience Edition (3 days / 3 zile): 18 ore / 18 CPD
```

These numbers are **not aligned** with the source documents, which propose:
- Online / Open cohort: 7 CPD
- On-site: 8 CPD
- Experience Edition: 12 CPD

The 6 / 12 / 18 values were placeholder assumptions in the generation script.

## 5. Programme → Edition Relationship

**Schema:** `Edition.programmeId` is non-nullable and has a single `Programme` relation. `@@unique([programmeId, slug])` prevents duplicate slugs within a programme.

**Code fix already applied:** `app/inscriere/page.tsx` and `app/calendar/page.tsx` now pass `programmeSlug` and filter editions by programme.

**Database verification:** cannot be run in this environment because `DATABASE_URL` is not configured. The validation script `scripts/validate-programme-edition-integrity.ts` exists and should be executed once the database connection is available.

## 6. CMS Data Model

Relevant Prisma fields:

```prisma
model Programme {
  availableDeliveryFormats DeliveryFormat[] @default([])
  duration                  String?
  learningHours             Int?
  cpdHours                  Int?
  // ...
}

enum DeliveryFormat {
  ONLINE
  ONSITE
  OPEN_COHORT
  EXPERIENCE_EDITION
}
```

**Observation:** the CMS stores a single `duration`, `learningHours` and `cpdHours` per programme. It does **not** natively store per-format values. If each format has different hours/credits, the CMS model needs either:
- a JSON field for format-specific hours/credits, or
- per-format records in a related table.

## 7. Recommendations

### Immediate

1. **Decide the canonical Experience Edition duration:**
   - If it is 3 days, update PMD, Commercial Sheet, CPD Package and website text to "3 days".
   - If it remains 2 days, ignore the user's note and keep current sources.

2. **Clarify "Learning Hours":**
   - Use "Contact hours" for PMD table values.
   - Use "Total learning hours" when including self-study.
   - Update PMDs and Commercial Sheets to use consistent labels.

3. **Align CPD values with source documents:**
   - Online: 7 CPD
   - On-site: 8 CPD
   - Open cohort: 7 CPD
   - Experience Edition: 12 CPD (or recalculate if 3 days)

4. **Fix programme order on website:**
   - Reorder `programs` array in `app/programe/page.tsx` to:
     1. Conversații care Contează
     2. AI Fără Haos
     3. Calm sub Presiune
     4. Busola Deciziilor
     5. Avantajul Uman

5. **Align delivery format names across all surfaces:**
   - Decide whether "Open cohort" is a customer-facing format.
   - Update PMD/Commercial Sheet wording to match website if needed.

### Structural

6. **Extend CMS model** to store per-format duration, learning hours and CPD credits if formats genuinely differ.

7. **Add data validation** in edition/programme forms to prevent cross-programme edition assignment.

8. **Re-run `scripts/validate-programme-edition-integrity.ts`** once `DATABASE_URL` is configured.
